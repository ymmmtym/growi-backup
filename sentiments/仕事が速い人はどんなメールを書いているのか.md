# 仕事が速い人はどんなメールを書いているのか <https://www.amazon.co.jp/gp/product/B06XQ1H6PZ> ## Table of contents ### はじめに - 仕事が早い人 = メールが早い人 ### １. 仕事が速い人のメールの発送 #### メールが速い人は仕事も速い？ - 単純に、メール処理にかかる時間を削減できる - 語彙力・文章力の高さを示せる #### 仕事が速い人の発想①「主導権」を握る - 仕事が早い人はメールに縛られていない - 相手に選択肢を提示して行動を限定する #### 仕事が速い人の発想② ムダなことを排除する - Next Actionの5W1Hを先に提示して、無駄なやり取りを事前に排除する #### 仕事が速い人の発想③ 優先順位をつけない - 即時対応してタスクを完了させる - 最低限やることの期日を伝える返答をする #### 仕事が速い人の発想④ 相手の思考を先回りする - 相手のレベルに合わせた対応をする #### 仕事が速い人が気をつけている ５つのポイント 1. 目的 2. ビジュアル 3. 返信しやすさ 4. 言葉 5. 処理時間の絵期限 ### ２. 目的を意識しながら書く #### すべてのビジネスメールには「目的」がある - 書いたメールが目的を達成できるものか考える #### 「目的」を伝えるために必要な６Ｗ＋３Ｈ - 以下の順序で、メールのやり取りを進める 1. who/what/why 2. when/where/whom 3. how to/how many/how much #### メールを送る前にシナリオをつくる - シナリオ：最初のアクションから作業終了までの間の具体的なステップ - 現在はどこのステップなのかを理解した上で、メールを送るタイミングや内容を考慮する #### 先回り力を駆使してメールを書く - 相手の行動を把握した上でメールをする - 相手が持ちそうな疑問について、あらかじめ補足する #### 一番効果的なタイミングでメールを送る - 概ねどのようなメールでも、**翌日**までには送る - 時間外や休日にはよほどのことがない限り対応しない(業務時間が守れてないイメージがつく) - 考慮が必要な場合は、検討の時間としてメールを3~5日遅らせる手もある #### 一度口にした約束を軽く扱わない - 約束を破ると信頼を損なう - 必ず実行できる事のみ、期限をつけて返答する #### 間違いを恐れないで送信する - 詳細チェックは時間の無駄。細かい点よりも目的が重要 - 機密情報を含むか、送信相手の信頼を損ねないかくらいは最低限確認 - 間違いは再度訂正すればいい ### ３. ビジュアルを工夫して読ませる #### 選ばれるのは読めるメール＆読みたいメール - 読みずらいメールの特徴 - 1行の文字数が多い（30文字以上） - 空白の行がない - 改行がない - 文章が5行以上続く - 箇条書きにすべき箇所が文章で書かれている - 段落/空白/読みさすさを意識する - 初見で嫌になるメールは送信者の問題 #### ブロック化・空行で本文を美しく整える - パッと見の印象がいい - ひらがなを効率よく使う #### 箇条書きで相手の理解度をアップさせる - 読者がどこに注意して読めばいいかわかる #### ７つの要素を意識してメールをつくる - 宛名/挨拶/名乗り/要旨/詳細/結び/署名 #### 「なぜ、このメールを送ったのか」を先に書く - 相手の理解してもらいやすくするため - 「結論 -> 詳細」の順で書く #### 文章をできるだけコンパクトにする - 1文を短くして分かりやすくする ### ４. 確実に返信を受け取る #### めんどくさそうなメールには返信をしてもらえない - "分かりやすさ"を追及してメールを送る #### 面識がない人に送るメールは〝ラブレター〟のように書く - 敬意を損ねない文章にする #### まずは自分から相手のファンになる - 返信をもらうために必要なことは、自分に"興味"を持ってもらうこと - 相手にも興味を持つ #### 件名に開封したくなるキーワードを入れる - 件名は簡潔に詳しく書く - 【緊急】【重要】などは、**本当に**必要なときに使う - 頻度が上がると読んでもらう確率が低くなる #### 「逃げ道」を用意して催促メールを送る - 相手から連絡がない時の催促 - こちらに非があった言い回しを使う - メールが遅れていたか心配であることを述べる #### 期限を１秒でも遅れたら問い合わせる - 期限を設けるときには、相手の都合を考慮 - 期限遅れに対して、すぐに問い合わせる理由 - 時間厳守という原則に価値を置いている - 遅れには断固とした態度で対応する - 1度でも許したことにより、次回以降も遅れるようにしないため #### 選択肢を示して返信を誘導する - 抽象的な質問をして、相手を困らせない - 選択肢や日時などは、明確に示す #### 相手の感情を大切に受け止める - メールでの感情は確実に拾って返答する - 相手の感情を受け止めた内容（同情）にする - お礼を述べる - 感謝の気持ちを伝える ### ５. 相手の心に刺さる言葉を使う #### 言葉で相手の心を動かす - メールの根幹は**言葉** #### 使えるフレーズはストックしておく - メールが上手な人は**表現力**に長けている - フレーズ例 - 「〇〇さんだからこそ、お願いしております。」：相手への信頼を寄せる - 「お力をお貸しください。」：ストレートにお願い - 「申し訳ありません。社の方針でして、、」：どうしても断りを入れるとき #### ネガティブフレーズをポジティブフレーズに変換する - 相手を全否定しないフレーズにする - こちらで出来る限りのことは対応する旨を記載する #### ムダな前置きはカットする - 長い前置きは不要 - 相手にとって重要なことは結論 #### 「お手すきのときに」という配慮は必要ない - 優先順位を判断するのは相手 - 期限を提示して相手が可能かどうか尋ねる #### 「させていただきます」を徹底排除する - 「させていただく -> 致します」で事足りる - 読み手に違和感を与えない #### 安易に「思います」を使わない - 自信の無さがうかがえる - あくまでも個人の気持ちを伝える時に使用する #### ３つのポイントでケンカ腰の相手をいなす - すぐに対処する - 怒りを最小限に抑える - 不用意に謝らない - 非を認めたことで、これ以上問題を発展させない - まずは**心配をかけたこと**について謝罪をする - 件名を変える - 自分がつけた件名を見て、相手の怒りが再燃しないように #### 聞きづらいことをあえて聞く - 対面では聞きづらいことを聞く - 可能な範囲で、差し支えなければ、、、、 - 教えてもらう理由も記載する - 次回からの参考にさせていただく #### 言葉のレベルを意識する - 依頼言葉のレベルについて（順軽度） 1. お手数ですが 2. ご面倒かとは存じますが 3. お手間を取らせて恐縮ですが 4. お手数をおかけしまして、はなはだ恐縮の至りではございますが - 違和感を与えない言葉のレベルを選ぶ ### ６. メールの処理時間を削減する #### ４つのアプローチで処理時間を減らす - 即レス - 処理時間 = (メールを読む時間 + メールを書く時間) README.md backend crud.py devops devops.md frontend growi.py index.md life.md network network.md others others.md pages.py requirements.txt sentiments user 通数 #### すべてのメールに即レスする - 受信メールの確認だけでも伝える #### 処理できないときはメールのチェックをしない - メールチェックは、朝、昼、夕方の3回で十分 #### 即レスできないメールはいったん受領の連絡をする - 受領したことを明確に伝える #### 部分引用を使って素早く返信する - 相手が問題と感じている箇所には部分的に回答する #### ＣＣは必要最低限にする - CCは人によってとらえ方が異なる場合がある - 目的の種類がが通知/報告などであるかをプロジェクトごとに事前に決める必要アリ #### あえてメール以外の連絡手段を使う - 電話を使った方がいいケース - 即答できる用件 - 緊急の用件 - 相手の理解を確認しながら説明するべき用件 - 感情的な要素が入るため、文章だけではトラブルになりやすい用件 #### 単語登録でスピーディーに入力する - 定型文を事前に登録しておく #### テンプレートを１２０％活用する - テンプレートを元に、相手の特徴などに合わせて文面を修正する(テンプレートと思わせない) #### キーワードを拾って速読する - 宛名/挨拶/名乗りを見極め、目的のみ読む
